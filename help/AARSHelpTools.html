Cool Capability: Plug-in Tools

Old tools in MARS 4.5 are selected and modified for usage in ARM. Just like MARS 4.5, AARS 0.2 is capable of running externally-developed software that meets a set of requirements. Everyone is encouraged to contribute new tools to this system. The requirements for such a program are:
  1. It implements the aars.tools.MarsTool interface.
  2. It is part of the aars.tools package.
  3. It compiles cleanly into a ".class" file, stored in the aars/tools directory.
AARS will detect all qualifying tools upon startup and include them in its Tools menu. When a tool's menu item is selected, an instance of it will be created using its no-argument constructor and its action() method will be invoked. If no qualifying tools are found at AARS startup, the Tools menu will not appear.
